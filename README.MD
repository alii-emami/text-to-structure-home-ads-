## 📖 درباره پروژه

**استخراج‌کننده اطلاعات آگهی‌های ملک** یک میکروسرویس آمادهٔ تولید (Production‑Ready) است که با دریافت متن یک آگهی ملک به زبان فارسی، اطلاعات کلیدی آن را به صورت یک خروجی JSON ساختاریافته برمی‌گرداند.

این اطلاعات شامل موارد زیر است:

- **متراژ** (مساحت واحد به متر مربع)
- **تعداد اتاق** (تعداد خواب یا اتاق)
- **وضعیت آسانسور** (دارد / ندارد / نامشخص)
- **وضعیت پارکینگ** (دارد / ندارد / نامشخص)
- **وضعیت بازسازی** (نوساز / بازسازی‌شده / قدیمی / نامشخص)

پروژه از دو رویکرد مکمل برای استخراج استفاده می‌کند:

1. **استخراج مبتنی بر قانون (Regex)** – برای الگوهای مشخص مانند متراژ (`۸۵ متر`) و تعداد اتاق (`دوخوابه`).
2. **مدل NER آموزش‌دیده (ParsBERT)** – برای درک بافت و تشخیص مفاهیم پیچیده‌تر مانند نفی (`آسانسور ندارد`) و عبارات ضمنی.

این ترکیب، دقت بالا در موارد رایج را با مقاومت در برابر تنوع زبانی در آگهی‌های فارسی تضمین می‌کند.

# 🐍 مثال استفاده با Python

```python
import requests

url = "http://127.0.0.1:8000/parse-ad"

payload = {
    "text": "واحد ۷۰ متری پارکینگ ندارد، آسانسور دارد"
}

response = requests.post(url, json=payload)

print(response.json())
```

# 🧠 نحوه عملکرد

## 1️⃣ پیش‌پردازش (Preprocessing)

متن ورودی با استفاده از `hazm.Normalizer` نرمال‌سازی می‌شود:

- تبدیل اعداد فارسی و عربی
- نرمال‌سازی علائم نگارشی
- اصلاح نیم‌فاصله‌ها
- حذف نویزهای متنی

---

## 2️⃣ استخراج با Regex

دو ویژگی اصلی با استفاده از عبارت‌های باقاعده استخراج می‌شوند.

### 📏 متراژ (`building_size`)

نمونه الگوهای قابل تشخیص:

- ۸۵ متر
- متراژ ۱۲۰
- زیربنا ۹۰

### 🛏️ تعداد اتاق (`rooms_count`)

نمونه الگوهای قابل تشخیص:

- ۲ خواب
- دوخوابه
- سه اتاق

---

## 3️⃣ استخراج با NER

برای استخراج ویژگی‌های معنایی از یک مدل آموزش‌دیده **ParsBERT NER** استفاده شده است.

### موجودیت‌های قابل تشخیص

| Label | Description |
|---------|------------|
| `ELEVATOR_POS` | دارای آسانسور |
| `ELEVATOR_NEG` | فاقد آسانسور |
| `PARKING_POS` | دارای پارکینگ |
| `PARKING_NEG` | فاقد پارکینگ |
| `REBUILT_POS` | بازسازی شده |
| `REBUILT_NEG` | بازسازی نشده |

### مثال

```text
آسانسور ندارد
```

⬇️

```text
ELEVATOR_NEG
```

مدل توانایی درک جملات مثبت و منفی را دارد و ویژگی‌ها را با توجه به بافت متن تشخیص می‌دهد.

---

## 4️⃣ تجمیع نتایج

در مرحله نهایی:

- خروجی Regex استخراج می‌شود.
- خروجی مدل NER استخراج می‌شود.
- نتایج با یکدیگر ادغام می‌شوند.
- پاسخ نهایی به‌صورت JSON بازگردانده می‌شود.

فیلدهایی که با اطمینان تشخیص داده نشوند مقدار `null` خواهند داشت.
